AI Becomes Physical

In manufacturing, AI doesn't just analyze — it inspects, controls, and decides. Mistakes have physical consequences.

Inspect

Computer vision catches defects humans miss. 24/7, consistent, sub-millimeter precision on production lines.

Control

RL optimizes temperature, pressure, speed in real time. Learns from trial and error without breaking things.

Decide

AI agents coordinate maintenance schedules, reroute production, escalate anomalies. Autonomous but supervised.

The stakes: A wrong recommendation in retail loses a sale. A wrong action on a production line can injure people or destroy equipment worth millions.

Week 7

Transfer Learning

Why Train From Scratch When You Can Adapt?

Deep learning needs massive data. Manufacturing often has only hundreds of images. Transfer learning solves this.

Without Transfer Learning

  • Need millions of labeled images
  • Training takes days/weeks on GPUs
  • Costs tens of thousands of dollars
  • Often impossible in specialized domains

With Transfer Learning

  • Start with a model trained on millions of images
  • Fine-tune with your 100–500 images
  • Training takes hours, not weeks
  • Often achieves 95%+ accuracy

Week 7

Transfer Learning in Practice

ImageNet (14M images, 20K categories) trained models are the starting point for most industrial vision.

Defect Detection

100 images of scratches on metal surfaces. Transfer from ResNet. 97% accuracy in detecting micro-cracks.

PCB Inspection

500 images of circuit boards. Transfer from EfficientNet. Detects solder defects faster than human inspectors.

Weld Quality

200 X-ray images of welds. Transfer from VGG. Classifies weld integrity: good, acceptable, reject.

Key takeaway: Transfer learning democratized deep learning. You don't need Google's data budget. You need a good pre-trained model and a few hundred labeled examples.

Week 7

Reinforcement Learning

Not supervised (no labels). Not unsupervised (there is a goal). The agent learns by doing and receiving rewards or penalties.

How It Works

  • Agent observes the state of the world
  • Takes an action
  • Receives a reward (or penalty)
  • Learns which actions maximize long-term reward

Real-World Examples

  • AlphaGo: learned to play Go better than any human
  • Robot arms: learned to grasp objects
  • HVAC systems: learned to minimize energy use
  • Data center cooling: Google saved 40% energy

State

What the agent observes. Temperature, pressure, vibration, production rate, time of day.

Action

What the agent can do. Adjust speed, change temperature, switch tooling, call maintenance.

Reward

Feedback signal. +1 for good output, -10 for defect, -100 for equipment damage, -1000 for safety incident.

Week 7

Reward Function Design

The reward function defines what "good" means. Get it wrong, and the agent optimizes for the wrong thing.

Good Reward Design

  • Maximize throughput AND minimize defects
  • Heavy penalty for safety violations
  • Reward maintenance that prevents breakdown
  • Balance multiple objectives

Bad Reward Design

  • "Maximize output" → agent skips quality checks
  • "Minimize cost" → agent defers all maintenance
  • "Zero defects" → agent runs so slowly nothing gets made
  • Reward hacking: agent finds loopholes

Goodhart's Law: "When a measure becomes a target, it ceases to be a good measure." Your reward function IS the product specification.

Week 7

Agent Architecture

Tool Use

Agents call external tools: databases, APIs, calculators, sensors, actuators. The LLM decides which tool to use and when.

Planning

Break complex goals into steps. "Inspect batch" becomes: capture image, run defect model, log results, flag if defective, update dashboard.

Memory

Short-term: current task context. Long-term: learned patterns, past incidents, equipment history. Agents improve over time.

Reflection

Agent evaluates its own output. "Did my action achieve the goal?" If not, it adjusts. Self-correction without human intervention.

Week 7

Agent Safety

In a factory, agent mistakes can injure people or destroy equipment. Safety is not optional.

Kill Switch — Human can override ANY agent action instantly
Escalation Logic — Agent knows when to ask for human help
Autonomy Boundaries — Clear limits on what the agent can do alone
Audit Trail — Every decision logged with reasoning

Rule of thumb: The higher the consequences of a wrong action, the lower the autonomy should be. Agents should earn trust gradually.

Decision: Which Base Model?

Transfer learning starts from a pre-trained model. Your choice matters.

Model Strengths Best For
ResNet Robust, well-understood, fast General defect detection
EfficientNet Best accuracy/efficiency ratio Edge deployment (on-device)
YOLO Real-time object detection Moving production lines
Vision Transformers Highest accuracy, attention-based Complex, subtle defects

Your decision: Match the model to your constraints. Edge device? EfficientNet. Real-time conveyor? YOLO. Maximum accuracy? Vision Transformer.

Week 7

Decision: Reward Function

What happens when you get the reward function wrong?

Scenario A: "Maximize Output"

Agent runs machines at maximum speed, skips quality checks, defers maintenance. Output goes up for 2 weeks. Then a $500K machine breaks.

Scenario B: "Balanced Reward"

+1 per unit produced. -5 per defective unit. -50 for unplanned downtime. -500 for safety incident. Agent learns to balance speed, quality, and safety.

The lesson: The reward function encodes your values. It's not a technical parameter — it's a business and ethical decision. Get stakeholders involved.

Week 7

Decision: When Should the Agent Escalate?

Agent Acts Alone

  • Routine quality pass/fail decisions
  • Small parameter adjustments within safe range
  • Standard maintenance scheduling
  • Data logging and reporting

Agent Escalates to Human

  • Anomaly it hasn't seen before
  • Action outside defined boundaries
  • Any safety-related decision
  • Equipment shutdown or production halt
  • Low confidence (<threshold) on any decision
Week 7

Decision: Agent Safety on a Line

A production line agent has real-world consequences. These safeguards are non-negotiable:

Technical Safeguards

  • Hard limits on all physical parameters
  • Watchdog timers: if agent stops responding, system halts
  • Redundant sensors: never trust a single reading
  • Graceful degradation: fallback to manual mode

Operational Safeguards

  • Shadow mode first: agent recommends, human decides
  • Gradual autonomy: earn trust with track record
  • Regular audits of agent decisions
  • Incident reporting and root cause analysis
Week 7

Modern AI: Edge AI & Human-Agent Collaboration

Edge AI

Run AI models on the device (camera, sensor, robot) instead of the cloud. Benefits: zero latency, works offline, data stays local. Critical for real-time factory decisions.

Human-Agent Collaboration

The future isn't fully autonomous factories. It's agents that amplify human workers: AI spots the defect, human decides the action. AI suggests the schedule, human approves.

The trend: Edge + Agents + Human-in-the-Loop = the industrial AI stack. Cloud for training, edge for inference, humans for judgment.
