## Why the split

Each file is self-contained for the phase or reference topic it covers. During class you open the file for the phase you are in; you do not scroll a 2,700-line single file looking for anchors. Across weeks (Week 6 media, Week 7 manufacturing, …) the same folder structure carries forward — the per-phase files' paste-ready blocks change; the README / appendix spine accretes.

The COC-level workflow prompts (`/analyze`, `/todos`, the four `/implement` sprints, `/redteam`, `/codify`) live in [`START_HERE.md`](./START_HERE.md) §6 — they are the workshop entries and are week-specific by design. The Playbook phase files here are the decisions you make inside `/implement`.
