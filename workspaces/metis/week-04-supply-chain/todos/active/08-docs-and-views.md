# Shard 08 — Docs and Views

## What

Author the three documents students interact with before and during the workshop, plus the two build scripts that keep them from drifting from their spec sources. `PLAYBOOK.md` is generated from the three playbook-phases specs; `SCAFFOLD_MANIFEST.md` is a generated view over `scaffold-contract.md`; `START_HERE.md` receives a final tactical pass resolving the five remaining HIGH findings from the red-team report (H5, H10, H11, H12, H13).

## Why

`PLAYBOOK.md` must never be hand-edited — its source is the three authoritative spec files, and hand edits create spec drift within a single session. The build script is the structural guard. `SCAFFOLD_MANIFEST.md` is read by the student's opening prompt (Claude Code verifies the scaffold against it instead of fabricating a list — Structural Issue 4 from the red-team report). The five pending H-findings in `START_HERE.md` are not cosmetic: H5 (missing grounding numbers) causes students to produce generic, ungradeable journal entries; H12 (Phase 11 only vs Phase 11+12) causes students to miss the re-solve step after union-cap fires and score 0/4 on D3.

## Implementation sketch

- `scripts/build_playbook.py` — reads the three spec files in order, prepends a "GENERATED — do not edit" comment + section headers, concatenates, writes `PLAYBOOK.md`; checks for any `TODO` marker in the input and raises with the source file + line number (prevents placeholder spec content from shipping in the student doc)
- `scripts/build_manifest.py` — parses `scaffold-contract.md` tables via regex on `| path |` rows, groups by section (§1–8), emits `SCAFFOLD_MANIFEST.md` with section headings + file state table + "last generated" timestamp; asserts every row has a non-empty state label
- `START_HERE.md` patches — targeted edits at 5 locations:
  - H5: Phase 2 + Phase 7 prompt templates: add "use the dollar figures in `specs/business-costs.md` — stockout: $40/unit, excess: $12/unit, SLA violation: $220/violation"
  - H10: add worked-examples subsection to rubric overview using rubric-grader.md §1.1-1.2 pair
  - H11: add D1 criterion annotation "$40 vs $12 asymmetry = 40:12 ratio named in units, not just 'costs more'"
  - H12: Sprint 2 scenario injection paragraph — spell out Phase 11 + Phase 12 re-run sequence explicitly
  - H13: add to §8 pitfalls: xgb import failure escape hatch + grader tolerance for 4-family leaderboard

## Out of scope

- L1–L6 low-severity findings (tone, capitalisation, attribution) — cosmetic, no grade impact
- M1–M12 medium findings deferred to /analyze per redteam-start-here-doc.md
- Any edits to the source spec files themselves (specs/ is the authority; this shard only builds views over it)
